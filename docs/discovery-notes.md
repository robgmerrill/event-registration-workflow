# Business Goals

The customer wants to reduce manual administrative work while improving visibility into upcoming event registrations.

The solution should be inexpensive, easy to maintain, and require minimal employee training.

---

# Current Workflow

1. Customer emails to register.
2. Event coordinator enters attendee into a spreadsheet.
3. Coordinator sends a manual confirmation email.
4. Operations manually tracks registration counts.
5. Managers request spreadsheet updates when they need reporting.

---

# Pain Points

- Duplicate manual work
- Confirmation emails are inconsistent
- Spreadsheet mistakes occur occasionally
- No real-time dashboard
- Difficult to answer questions like:
  - How many registrations do we have?
  - Which events are most popular?
  - Has everyone received confirmation?

---

# Constraints

- Small budget
- Small operations team
- No dedicated IT staff
- Prefer SaaS products over managing servers
- Solution should be deployable in a few days

---

# Questions Asked

### Where should registration data live?

The customer is comfortable using Airtable because it provides a familiar spreadsheet interface while offering an API.

### Who needs access?

Operations staff only.

Attendees do not require accounts.

### What information is required?

- Attendee Name
- Email Address
- Event Name
- Registration Status

---

# Success Metrics

- Reduce manual registration work.
- Standardize confirmation messaging.
- Improve operational visibility.
- Enable future workflow automation.
